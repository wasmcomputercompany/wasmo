Email Address Linking
=====================

Linking an email address to an account is necessary and sufficient to sign up or sign in.


Happy Path
----------

In the happy path, the user calls this API:

```
POST /link-email-address HTTP/1.1
content-type: application/json

{
  "unverifiedEmailAddress": "jesse@example.com"
}
```

The server responds with a token:

```
HTTP/1.1 200 OK
content-type: application/json

{
  "challengeToken": "PWxtnQkLOKZZt2SUDxFnqXnXH.PWxtnQkLOKZZt2ESUD+FnqXnXHJbIpPX5s4/2IWOPjY"
}
```

The user checks their email, and makes a follow-up call with the emailed token:

```
POST /confirm-email-address HTTP/1.1
content-type: application/json

{
  "challengeToken": "PWxtnQkLOKZZt2SUDxFnqXnXH.PWxtnQkLOKZZt2ESUD+FnqXnXHJbIpPX5s4/2IWOPjY",
  "code": "123456"
}
```

The server links the email address and responds with an account snapshot.

```
HTTP/1.1 200 OK
content-type: application/json

{
  "success": true,
  "account: {
    (updated account snapshot data)
  }
}
```

The Challenge Token
-------------------

Our inputs:

- A 25-character randomly-generated salt, like `PWxtnQkLOKZZt2SUDxFnqXnXH`.
- The account ID, like `55`.
- The email to be linked, like `jesse@example.com`.
- A randomly-generated token, like `123456`

We concatenate these values, separated by newlines to create the HMAC input.

```
PWxtnQkLOKZZt2SUDxFnqXnXH
55
jesse@example.com
123456
```

The challenge token is the salt, followed by a `.` followed by the HMAC.

We use the service's cookie secret as the HMAC key.

We can perform this issuing process to check a challenge attempt.


Rate Limiting
-------------

We use our [permitting system](./rate_limiting_permits.md) to limit email link attempts.

For example, for email address linking we insert the key with type `EmailAddressLink` and value
`jesse@example.com` on each attempt.


Effects of Linking
------------------

There's no customer merging.

If the email address is already linked to an existing account ID, linking it will change the
caller's cookie to point to this existing account ID (signing in).

If the email address is not already linked to an existing account ID, linking it will attach it to
the caller's account (signing up).


The EmailAddressLink Table
--------------------------

We link email addresses to accounts.

```
CREATE TABLE EmailAddressLink (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMPTZ NOT NULL,
  account_id BIGINT NOT NULL,
  email_address VARCHAR(255) UNIQUE NOT NULL,
  active BOOLEAN NULL
);

CREATE UNIQUE INDEX EmailAddressLinkEmailAddressActive ON EmailAddressLink (
  email_address,
  active
) WHERE active IS NOT NULL;
```
