Rate Limiting Permits
=====================

We have a general-purpose rate limiting system.

```
CREATE TABLE Permit (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  type VARCHAR(255) NOT NULL,
  value VARCHAR(255) NOT NULL,
  serial_number BIGINT,
  acquire_at TIMESTAMPTZ  NOT NULL
);

CREATE UNIQUE INDEX PermitTypeValueSerialNumber
ON Permit (
   type,
   value,
   serial_number
);
```

Each time a customer attempts a rate-limited action (like linking an email address), we track that.

We uniquely index serial numbers to detect and prevent races. Two processes that attempt to acquire
the same permit at the same time will collide when inserting into the unique index. This forces
an opportunity to fail or retry.
